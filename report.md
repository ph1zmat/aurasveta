# SEO-аудит aurasveta.by: детальный отчёт и план действий

## Краткое резюме (Executive Summary)

Сайт **aurasveta.by** — это интернет-магазин светотехники в Мозыре (Беларусь), построенный на современном стеке (Next.js 15, App Router, SSR/ISR). Технически SEO реализовано **качественно**: динамическая генерация sitemap из БД, корректная Schema.org разметка (Product, Organization, WebSite, BreadcrumbList), система управления метаданными через админку с возможностью ручного переопределения автоматических SEO-полей, Open Graph и Twitter Cards. Однако сайт сталкивается с **критическими проблемами**, блокирующими рост органического трафика: полное отсутствие в индексе Google, несоответствие контента белорусскому рынку (российские рубли, упоминания Москвы и СберСпасибо), отсутствие информационного контента и слабая конкурентная позиция против крупных игроков (21vek.by, deal.by).

**Прогноз**: при выполнении всех критических и важных задач возможен выход в топ-10 по геозависимым запросам (Мозырь + светотехника) через 3–6 месяцев. На национальные запросы потребуется 12+ месяцев.

## Этап 2. Технический аудит

### 2.1. Robots.txt и Sitemap

**Robots.txt** (`app/robots.ts`):
```typescript
export default function robots(): MetadataRoute.Robots {
  return {
    rules: [{ userAgent: '*', allow: '/' }],
    sitemap: 'https://aurasveta.by/sitemap.xml',
  }
}
```

**Оценка**: ✅ Корректно. Разрешён весь сайт, указан sitemap.

**Sitemap** (`app/sitemap.ts`): Генерируется динамически из БД (Prisma), включает:
- Статические страницы (/, /catalog) — priority 1.0 и 0.9
- Категории (`/catalog/[slug]`) — priority 0.7
- Товары (`/product/[slug]`) — priority 0.8
- CMS-страницы (`/pages/[slug]`) — priority 0.5

**Проблемы sitemap**:
- ❌ **Нет `<image:image>`** для товарных страниц — Google может не индексировать изображения товаров
- ❌ **changefrequency устаревший сигнал** — Google его игнорирует, но для Яндекса полезен
- ⚠️ **Нет альтернативных языков** — если планируется i18n, нужен `hreflang`

### 2.6. Архитектура SEO-системы

Система управления SEO реализована на **высоком уровне**:

**Файлы**:
- `lib/seo/getMetadata.ts` — получение метаданных (auto + override из БД)
- `lib/seo/metadata-persistence.ts` — сохранение/слияние SEO-полей
- `shared/lib/seo/generateSeo.ts` — генерация автоматических title/description
- `shared/lib/seo/snippet-trial-summary.ts` — анализ сниппетов
- `shared/ui/ProductStructuredData.tsx` — JSON-LD для товаров
- `shared/ui/BreadcrumbStructuredData.tsx` — JSON-LD для хлебных крошек

**Механизм**:
1. Автоматическая генерация SEO на основе названия товара/категории
2. Возможность ручного переопределения через админку (таблица `seoMetadata` в Prisma)
3. Гибридный подход: `merge(auto, override)` — ручные поля имеют приоритет

**Оценка**: ✅ Продвинутая система, редко встречается в проектах такого масштаба.

### ❌ 3. Блога НЕ будет (политика project)

**Решение**: информационный контент создаётся через существующую CMS (`/pages/[slug]`), **без статического хардкода**.

**Процесс**:
1. Редактор создаёт страницу в админке (`/admin/pages`)
2. Устанавливает slug (например `kak-vybrat-lyustru`)
3. Добавляет SEO-поля через встроенную систему
4. Публикует → страница доступна как `/pages/kak-vybrat-lyustru`
5. Система автоматически добавляет в sitemap

Преимущества:
- Нет моков в коде
- Управляется через админку
- Масштабируется любое количество статей
- SEO система единая для всех типов контента
